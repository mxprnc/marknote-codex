# 인터랙티브 TUI 사용

Codex CLI의 기본 사용 방식은 전체 화면 터미널 UI(TUI)입니다. 저장소를 읽고, 파일을 수정하고, 명령을 실행하면서 Codex와 대화하듯 반복 작업할 수 있습니다.

## 시작하기

기본 실행:

```bash
codex
```

초기 프롬프트와 함께 실행:

```bash
codex "Explain this codebase to me"
```

한국어로도 자연스럽게 사용할 수 있습니다.

```bash
codex "이 저장소의 구조와 주요 실행 경로를 설명해줘"
```

## TUI에서 할 수 있는 일

인터랙티브 세션이 열리면 다음 작업을 한 화면에서 처리할 수 있습니다.

- 프롬프트, 코드 조각, 스크린샷을 입력창에 직접 넣기
- Codex가 제안하는 계획을 읽고, 수정 전후의 동작을 실시간으로 확인하기
- 문법 강조가 적용된 Markdown 코드 블록과 diff 확인하기
- `/theme`로 TUI 테마를 미리 보고 저장하기
- `/clear`로 터미널을 지우고 새 대화 흐름 시작하기
- `/copy` 또는 `Ctrl+O`로 최근 완료된 Codex 출력 복사하기
- Codex가 작업 중일 때 `Tab`을 눌러 다음 턴에 실행할 후속 입력을 큐에 넣기
- 입력창에서 `Up`/`Down`으로 이전 초안 기록 탐색하기
- `Ctrl+R`로 프롬프트 기록 검색하기
- `Ctrl+C` 또는 `/exit`로 세션 종료하기

## 입력 흐름과 단축키

| 동작 | 방법 |
| --- | --- |
| 화면만 지우기 | `Ctrl+L` |
| 대화 흐름을 새로 시작하기 | `/clear` |
| 최근 완료 출력 복사 | `/copy` 또는 `Ctrl+O` |
| Codex 작업 중 현재 턴에 지시 추가 | `Enter` |
| Codex 작업 중 다음 턴 입력 예약 | `Tab` |
| 이전 초안 불러오기 | `Up` / `Down` |
| 프롬프트 기록 검색 | `Ctrl+R` |
| 검색 결과 적용 | `Enter` |
| 검색 취소 | `Esc` |
| 세션 종료 | `Ctrl+C` 또는 `/exit` |

`/copy`는 현재 턴이 아직 실행 중이면 진행 중인 텍스트가 아니라 가장 최근에 완료된 Codex 출력을 복사합니다. 따라서 긴 작업 중에도 안정적으로 직전 결과를 가져올 수 있습니다.

## 파일 경로 빠르게 입력하기

입력창에서 `@`를 입력하면 워크스페이스 루트 기준의 fuzzy 파일 검색이 열립니다. 원하는 파일을 고른 뒤 `Tab` 또는 `Enter`를 누르면 해당 경로가 프롬프트에 삽입됩니다.

예를 들어 특정 파일을 기준으로 리뷰를 요청할 수 있습니다.

```text
@src/auth/session.ts 이 파일의 세션 만료 처리 흐름을 설명해줘
```

## 로컬 쉘 명령 실행

입력 줄 앞에 `!`를 붙이면 로컬 쉘 명령을 실행할 수 있습니다.

```text
!ls
```

Codex는 이 명령의 출력을 사용자가 제공한 명령 결과처럼 취급합니다. 단, 승인 모드와 샌드박스 설정은 그대로 적용됩니다.

## 이전 메시지 수정과 fork

입력창이 비어 있을 때 `Esc`를 두 번 누르면 직전 사용자 메시지를 수정할 수 있습니다. 계속 `Esc`를 누르면 더 이전 메시지로 이동할 수 있고, `Enter`를 누르면 그 시점에서 대화를 fork합니다.

이 기능은 “아까 요청을 조금 다르게 했어야 했다”는 상황에서 유용합니다. 전체 세션을 다시 시작하지 않고 이전 지점으로 돌아가 다른 방향으로 진행할 수 있습니다.

## 프롬프트 편집기

긴 프롬프트를 작성할 때는 입력창에서 `Ctrl+G`를 눌러 외부 편집기를 열 수 있습니다. Codex는 `VISUAL` 환경 변수를 먼저 사용하고, 없으면 `EDITOR`를 사용합니다.

예:

```bash
export VISUAL=vim
codex
```

TUI 입력창에서 `Ctrl+G`를 누르면 Vim에서 프롬프트를 작성한 뒤 다시 Codex 입력창으로 보낼 수 있습니다.

## 테마와 문법 강조

TUI는 fenced Markdown 코드 블록과 파일 diff를 문법 강조로 보여줍니다. 코드 리뷰나 디버깅 중 긴 출력을 훨씬 쉽게 읽을 수 있습니다.

테마 선택:

```text
/theme
```

`/theme` 명령을 실행하면 테마 선택기가 열리고, 테마를 실시간으로 미리 본 뒤 선택 값을 저장할 수 있습니다. 저장 위치는 기본적으로 `~/.codex/config.toml`의 `tui.theme`입니다.

커스텀 테마를 쓰려면 `.tmTheme` 파일을 `$CODEX_HOME/themes` 아래에 넣고 테마 선택기에서 고르면 됩니다.
